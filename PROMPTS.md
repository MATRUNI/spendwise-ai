# 📝 LLM Prompts & Engineering

This document outlines the prompt engineering strategy used in Spendwise AI to power the **"Wise Engine"** rationales.

## 1. The Core Rationale Prompt
This prompt is used to generate the qualitative explanations for each financial recommendation.

### The Prompt
```text
You are an expert SaaS auditor. Analyze this AI software stack and write a professional 2-sentence rationale for each recommendation.
Focus on qualitative logic, not the math. Return ONLY a JSON array of strings.

Recommendations:
[
  {
    "tool": "Claude Pro",
    "action": "Cancel",
    "reason": "Redundant with ChatGPT Plus seat"
  },
  ...
]
```

### Why we wrote it this way:
*   **Persona Grounding:** Starting with *"You are an expert SaaS auditor"* ensures the tone is authoritative and professional (B2B SaaS style) rather than generic AI chatter.
*   **Math Guardrails:** We explicitly state *"Focus on qualitative logic, not the math."* This is because the math is already handled by our `auditEngine.js`. LLMs are notorious for hallucinating arithmetic; we use the LLM for the **"Why"**, not the **"How Much"**.
*   **Structural Constraint:** *"Return ONLY a JSON array of strings"* paired with Gemini's `responseMimeType: "application/json"` flag ensures the response can be directly parsed and mapped to our UI components without regex cleaning.

## 2. What didn't work (Iterative Failures)

### ❌ The "Single-Call Analysis"
**Attempt:** We tried sending the entire user profile and asking the AI to "Find savings and explain them."
**Failure:** The AI would often miss the subtle redundancy rules (like identifying that a Cursor subscription includes Claude/GPT-4 access). It also struggled to remain consistent with our hard-coded pricing data.
**Pivot:** We moved the logic to a deterministic Javascript engine (`auditEngine.js`) and used the AI only to "narrate" the results.

### ❌ The "Conversation" Trap
**Attempt:** Initially, the prompt didn't specify "ONLY JSON."
**Failure:** The LLM would prepend text like *"Here are your audit results:"* or append helpful tips at the end. This crashed the JSON parser on the frontend.
**Pivot:** Implemented strict schema enforcement and used the `responseMimeType` configuration.

### ❌ Relying on Anthropic only
**Attempt:** Initially used Claude 3 Haiku via a single endpoint.
**Failure:** In production, we hit frequent 401/404 errors due to regional restrictions and API quota limits.
**Pivot:** Developed the **"Wise Fallback Chain"** which attempts the prompt across Gemini 2.0 → Gemini 1.5 → Gemma to ensure 99.9% reliability.

## 3. Data Grounding
To prevent the AI from hallucinating features, we pass the `fallbackRationale` generated by our engine into the prompt.
*   **Example:** Instead of asking the AI to guess why we are canceling a tool, we tell it: *"We are canceling Tool X because of Rule Y. Explain the strategic value of this."*
*   **Result:** The AI sounds smarter because it’s "narrating" a high-quality data point provided by the system.
